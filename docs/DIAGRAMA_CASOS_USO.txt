DIAGRAMA DE CASOS DE USO - TECHSTORE
====================================

ACTOR PRINCIPAL: Usuario (Cliente)

CASOS DE USO:

1. REGISTRARSE
   - Actor: Usuario no registrado
   - Precondición: Usuario no tiene cuenta
   - Flujo principal:
     1. Usuario ingresa a la pantalla de registro
     2. Completa formulario (nombre, email, teléfono, contraseña)
     3. Sistema valida datos
     4. Sistema crea cuenta en base de datos
     5. Sistema inicia sesión automáticamente
     6. Usuario es redirigido a productos
   - Postcondición: Usuario tiene cuenta activa

2. INICIAR SESIÓN
   - Actor: Usuario registrado
   - Precondición: Usuario tiene cuenta
   - Flujo principal:
     1. Usuario ingresa email y contraseña
     2. Sistema valida credenciales en base de datos
     3. Sistema inicia sesión
     4. Usuario es redirigido a productos
   - Postcondición: Usuario autenticado

3. RECUPERAR CONTRASEÑA
   - Actor: Usuario registrado
   - Precondición: Usuario olvidó su contraseña
   - Flujo principal:
     1. Usuario ingresa su email
     2. Sistema valida que el email existe
     3. Sistema simula envío de enlace de recuperación
     4. Usuario recibe confirmación
   - Postcondición: Usuario puede recuperar acceso

4. VER PRODUCTOS
   - Actor: Usuario autenticado
   - Precondición: Usuario ha iniciado sesión
   - Flujo principal:
     1. Usuario accede a lista de productos
     2. Sistema muestra productos en RecyclerView
     3. Usuario puede ver detalles de cada producto
   - Postcondición: Usuario visualiza catálogo

5. CREAR PRODUCTO
   - Actor: Usuario autenticado
   - Precondición: Usuario ha iniciado sesión
   - Flujo principal:
     1. Usuario hace clic en botón "Agregar"
     2. Sistema muestra formulario de producto
     3. Usuario completa datos (nombre, descripción, precio, categoría, stock)
     4. Sistema valida datos
     5. Sistema guarda producto en base de datos
     6. Producto aparece en la lista
   - Postcondición: Nuevo producto disponible

6. EDITAR PRODUCTO
   - Actor: Usuario autenticado
   - Precondición: Producto existe
   - Flujo principal:
     1. Usuario selecciona producto
     2. Usuario hace clic en "Editar"
     3. Sistema muestra formulario con datos actuales
     4. Usuario modifica datos
     5. Sistema valida y actualiza en base de datos
   - Postcondición: Producto actualizado

7. ELIMINAR PRODUCTO
   - Actor: Usuario autenticado
   - Precondición: Producto existe
   - Flujo principal:
     1. Usuario selecciona producto
     2. Usuario hace clic en "Eliminar"
     3. Sistema solicita confirmación
     4. Usuario confirma
     5. Sistema elimina de base de datos
   - Postcondición: Producto eliminado

8. AGREGAR AL CARRITO
   - Actor: Usuario autenticado
   - Precondición: Usuario autenticado y producto disponible
   - Flujo principal:
     1. Usuario selecciona producto
     2. Usuario hace clic en "Agregar al carrito"
     3. Sistema verifica stock
     4. Sistema agrega item al carrito en base de datos
     5. Sistema muestra confirmación
   - Postcondición: Producto en carrito

9. VER CARRITO
   - Actor: Usuario autenticado
   - Precondición: Usuario tiene items en carrito
   - Flujo principal:
     1. Usuario accede a carrito
     2. Sistema muestra items en RecyclerView
     3. Usuario puede modificar cantidades
     4. Sistema calcula total
   - Postcondición: Usuario visualiza carrito

10. MODIFICAR CANTIDAD EN CARRITO
    - Actor: Usuario autenticado
    - Precondición: Item existe en carrito
    - Flujo principal:
      1. Usuario hace clic en aumentar/disminuir
      2. Sistema actualiza cantidad en base de datos
      3. Sistema recalcula subtotal
      4. Sistema actualiza total
    - Postcondición: Cantidad actualizada

11. ELIMINAR DEL CARRITO
    - Actor: Usuario autenticado
    - Precondición: Item existe en carrito
    - Flujo principal:
      1. Usuario hace clic en "Eliminar"
      2. Sistema elimina item de base de datos
      3. Sistema actualiza lista y total
    - Postcondición: Item eliminado

12. REALIZAR COMPRA
    - Actor: Usuario autenticado
    - Precondición: Carrito tiene items
    - Flujo principal:
      1. Usuario revisa carrito
      2. Usuario hace clic en "Realizar Compra"
      3. Sistema solicita confirmación
      4. Usuario confirma
      5. Sistema limpia carrito
      6. Sistema muestra confirmación
    - Postcondición: Compra realizada

13. VACIAR CARRITO
    - Actor: Usuario autenticado
    - Precondición: Carrito tiene items
    - Flujo principal:
      1. Usuario hace clic en "Vaciar Carrito"
      2. Sistema solicita confirmación
      3. Usuario confirma
      4. Sistema elimina todos los items
    - Postcondición: Carrito vacío

14. CERRAR SESIÓN
    - Actor: Usuario autenticado
    - Precondición: Usuario tiene sesión activa
    - Flujo principal:
      1. Usuario accede a perfil
      2. Usuario hace clic en "Cerrar Sesión"
      3. Sistema limpia datos de sesión
      4. Usuario es redirigido a pantalla de bienvenida
    - Postcondición: Sesión cerrada

RELACIONES ENTRE CASOS DE USO:
-------------------------------
- REGISTRARSE → INICIAR SESIÓN (automático)
- INICIAR SESIÓN → VER PRODUCTOS
- VER PRODUCTOS → AGREGAR AL CARRITO
- VER PRODUCTOS → CREAR PRODUCTO
- VER PRODUCTOS → EDITAR PRODUCTO
- VER PRODUCTOS → ELIMINAR PRODUCTO
- AGREGAR AL CARRITO → VER CARRITO
- VER CARRITO → MODIFICAR CANTIDAD
- VER CARRITO → ELIMINAR DEL CARRITO
- VER CARRITO → REALIZAR COMPRA
- VER CARRITO → VACIAR CARRITO
